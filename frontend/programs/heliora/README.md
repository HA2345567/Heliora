# Heliora Anchor program

Solana smart contract powering the Heliora prediction-market protocol.

## What's here

| File | Purpose |
|---|---|
| `programs/heliora/src/lib.rs` | Program: market factory, LMSR AMM, SPL position mints, resolve/claim, agent registry |
| `Anchor.toml` | Devnet/localnet program ids, provider |
| `Cargo.toml` (workspace) | Anchor workspace |

## Instructions

| Ix | Purpose |
|---|---|
| `initialize_protocol` | One-time bootstrap of `ProtocolConfig` PDA |
| `create_market` | Deploys a binary market, mints YES/NO SPL mints, opens collateral vault |
| `buy` | LMSR cost → trader pays USDC, receives SPL position tokens |
| `resolve` | Authority/creator sets outcome (Pyth/Switchboard/AI-Oracle resolved off-chain) |
| `claim` | Burn winning SPL position, receive 1:1 USDC from vault |
| `register_agent` | First-class on-chain agent identity |

## Build & deploy (do this yourself — Lovable cannot)

```bash
# Toolchain
sh -c "$(curl -sSfL https://release.anza.xyz/stable/install)"
cargo install --git https://github.com/coral-xyz/anchor avm --force
avm install 0.30.1 && avm use 0.30.1

# Build
anchor build

# Get the real program id from the keypair generated by anchor build
solana address -k target/deploy/heliora-keypair.json
# → paste it into Anchor.toml [programs.devnet]
# → and into programs/heliora/src/lib.rs `declare_id!(...)`
anchor build               # rebuild with the real id

# Deploy to devnet
solana airdrop 2
anchor deploy --provider.cluster devnet
```

After deploy, copy the program id into the backend env (`HELIORA_PROGRAM_ID`)
and the frontend env (`VITE_HELIORA_PROGRAM_ID`).

## Notes

- The on-chain LMSR uses a Taylor approximation of `exp/ln` for compactness.
  For mainnet replace with a verified soft-float crate (`fixed`, `substrate-fixed`, etc.) and re-audit.
- The vault is a USDC ATA owned by the `market` PDA. Mint USDC mock on devnet
  with `spl-token create-token` for testing.
- Resolution authority is currently `config.authority` or market creator. The AI-Oracle
  off-chain consensus signs the `resolve` instruction with the configured keypair.
- Three independent audits required before mainnet (per PRD §12).
